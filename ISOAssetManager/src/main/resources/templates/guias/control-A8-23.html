CONTROL El acceso a sitios web externos debería gestionarse para reducir la exposición a contenido malintencionado.
OBJETIVOS Para proteger los sistemas contra el peligro de malware y para evitar el acceso a recursos no autorizador de la web.
GUÍAS

La organización debería reducir los riesgos de que su personal acceda a sitios web que contienen información ilegal o que se sabe que contienen virus o material de phishing. Una técnica para lograr esto funciona al bloquear la dirección IP o el dominio de los sitios web en cuestión. Algunos navegadores y tecnologías antimalware lo hacen automáticamente o pueden configurarse para hacerlo.

La organización debería identificar los tipos de sitios web a los que el personal debería o no tener acceso. La organización debería considerar bloquear el acceso a los siguientes tipos de sitios web:

  1. Sitios web que tengan una función de carga de información a menos que se permita por razones comerciales válidas;
  2. Sitios web maliciosos conocidos o sospechosos (por ejemplo, aquellos que distribuyen contenido de malware o phishing);
  3. Servidores de comando y control;
  4. Sitio web malicioso adquirido de inteligencia contra amenazas (ver 5.7);
  5. Sitios web que comparten contenido ilegal.

Antes de implementar este control, la organización debería establecer reglas para un uso seguro y apropiado de recursos en línea, incluida cualquier restricción a sitios web indeseables o inadecuados y aplicaciones basadas en web. Las normas deberían mantenerse actualizadas.

Se debería impartir capacitación al personal sobre el uso seguro y adecuado de los recursos en línea, incluido el acceso a la Web. La formación debería incluir las reglas de la organización, el punto de contacto para plantear problemas de seguridad y el proceso de excepción cuando se necesite acceder a recursos web restringidos por razones empresariales legítimas. También se debería impartir formación al personal para asegurarse de que no anulan ningún aviso del navegador que informe de que un sitio web no es seguro, pero que permite al usuario continuar.

EJEMPLOS DE IMPLEMENTACIÓN

Internet es un campo minado. El filtrado web actúa como un detector de metales para evitar que el personal pise donde no debe:

Ejemplo 1: Bloqueo de Servidores de Comando y Control (C2) a nivel DNS

La empresa utiliza un servicio de resolución DNS seguro (como Cisco Umbrella o Quad9). Incluso si un empleado hace clic accidentalmente en un enlace de phishing y se descarga un malware, cuando ese malware intenta "llamar a casa" al servidor del atacante para recibir instrucciones, la petición DNS es bloqueada automáticamente, neutralizando el ataque y alertando al equipo de seguridad.

Ejemplo 2: Prevención de Fuga de Datos (Shadow IT)

Para evitar que información confidencial salga de la empresa sin control, el proxy web corporativo está configurado para bloquear el acceso a categorías como "Almacenamiento en la Nube Personal" (Dropbox, Google Drive personal, WeTransfer) y "Webmail Personal". Solo se permite el acceso a las instancias corporativas autorizadas (por ejemplo, el SharePoint de la empresa), previniendo así la exfiltración de documentos sensibles.
OTRA INFO. El filtrado web puede incluir una serie de técnicas, entre las que se incluyen firmas, heurística, lista de sitios web o dominios aceptables, lista de sitios web o dominios prohibidos y configuración personalizada para ayudar a evitar que software malintencionado y otras actividades maliciosas ataquen la red y los sistemas de la organización.
